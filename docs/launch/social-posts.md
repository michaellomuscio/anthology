### Option A — Builder narrative (recommended for LinkedIn)

> ✨ I built the tool I wished existed.
>
> I love Claude Code. The agentic coding workflow changed how I ship software. But the moment my single Claude session became four — one polishing tests, one drafting a PR, one exploring a refactor, one chasing a bug — my terminal couldn't keep up.
>
> Tabs got lost. Scrollback disappeared on a crash. I missed the moment a session went silent waiting on a permission decision.
>
> Existing terminal managers are nice. None were built for this specific shape of work.
>
> So I built one.
>
> Meet **Anthology** — a native macOS app for orchestrating many Claude Code sessions at once.
>
> 🐝 Real PTYs (genuine terminals, not mocks)
> 🐝 Persistent scrollback that survives crashes
> 🐝 Status dots — running, idle, waiting, error
> 🐝 Mission Control grid for watching the whole fleet
> 🐝 "Project Manager" mode where one Claude coordinates the others via MCP
>
> Then I realized I wanted to check on my agents from anywhere — without dragging my MacBook around.
>
> So I also built **Anthology iOS**. An iPhone + iPad companion that pairs with the Mac in two taps (scan a QR code) and lets me view, type into, and steer every session remotely. With push notifications when one needs me.
>
> Both apps are free. Both are open source under Apache 2.0. Apple Silicon Macs and iOS 17+.
>
> If you run multiple Claude Code sessions, give it a look:
> 👉 github.com/michaellomuscio/anthology
>
> Built on weekends and lunch breaks. Sharing because I'm probably not the only one juggling more agents than terminals.
>
> #ClaudeCode #Anthropic #DeveloperTools #OpenSource #MacOS #iOS #BuildInPublic #AgenticAI

## Posting cadence (suggested)

If you want to milk the launch for a week of content:

| Day | Platform | Post |
|---|---|---|
| **Mon** (day -1) | Instagram Story | Sneak peek of the bee + "tomorrow" |
| **Tue** (launch) | LinkedIn (Option A) | Big builder narrative + hero image |
| **Tue** (launch) | Instagram (Option A) | Same story, carousel format |
| **Tue** (launch) | Twitter/X | Tight version + link |
| **Wed** (+1) | LinkedIn comment | "24 hours in: X stars, Y downloads, here's what surprised me" |
| **Thu** (+2) | Instagram | Behind-the-scenes: "what I built it with" (Electron, SwiftUI, Cloudflare Workers, MCP) |
| **Fri** (+3) | LinkedIn | Lessons learned post — "5 things I learned building a Mac + iOS + Worker stack solo" |
| **Sun** (+5) | Substack `fullstackeducator` | Long-form: full story of why I built it, what I learned, what's next |

Each post recycles the visuals from the launch kit but pairs different prose. Stretches one launch event across a full week of content.
